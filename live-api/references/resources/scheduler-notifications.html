Notifications du planificateur

Lors de la création de flux de travail avec le planificateur, vous pouvez éventuellement configurer des notifications. Après avoir terminé une tâche, le planificateur enverra une notification au point de terminaison configuré à l'aide de la méthode http PUBLIER avec des détails sur l'action qui a été effectuée et son succès/échec. Vous pouvez également configurer une pré-notification d'une tâche à venir. Veuillez vous référer au Référence API pour configurer les notifications pour votre type de flux de travail.

Corps de notification

Lorsque le serveur envoie une notification à votre notification, la demande inclut un corps JSON

Champ Description
action Nom de l'action. Voir Actions
workflow Mise à jour du flux de travail une fois l'action terminée. La structure dépendra du type de flux de travail.
erreur Si une erreur s'est produite lors de l'exécution de l'action, ce champ contiendra l'erreur. la valeur dépend du type d'action.

Actions

Prénom Type de flux de travail Description
notifier-activer jobstartstop Attention, votre Job sera bientôt activé.
activer jobstartstop Activation de la tâche terminée. Si "Erreur" est présent, l'activation a échoué et la désactivation sera annulée.
notifier-désactiver jobstartstop Attention, votre Job sera bientôt activé.
désactiver jobstartstop Désactivation de l'emploi forcée. Si "Erreur" est présent, la désactivation a échoué.
notification-clip agrafe Attention, votre clip sera bientôt demandé.
agrafe agrafe Demande de clip terminée. Si "Erreur" est présent, la demande a échoué.
