DRM en direct via l'API Live

Cette rubrique traite de l'ajout de DRM pour les flux en direct via l'API Live.

Introduction

La gestion des données DRM peut être ajoutée aux tâches en cours en incluant l' DRM objet dans le corps de la requête lorsque vous créez le travail. L' DRM objet possède deux propriétés :

  • modes défini sur un tableau ou un ou plusieurs des éléments suivants : « all », « fairplay », « playready », « widevine », « widevine:dash » ou « widevine:hls »
  • token_expires_in défini sur un nombre compris entre 43200 (par défaut) et 31622400 (le nombre de secondes avant l'expiration du jeton)

La définition mode de tous entraînera l'application de tous les types de DRM disponibles aux sorties. Par défaut, DRM est appliqué aux sorties comme suit :

  • FairPlay : appliqué aux sorties HLS
  • Playready : appliqué aux sorties DASH
  • Widevine : appliqué aux sorties HLS

Widevine peut être appliqué aux sorties DASH au lieu de HLS. Pour faire de Widevine avec DASH la valeur par défaut nécessite un remplacement au niveau du compte. Contactez le support technique si vous souhaitez que cette dérogation soit appliquée à votre compte. Dans la section suivante, vous verrez comment vous pouvez spécifier widevine:hls ou widevine:dash pour des travaux particuliers, quelle que soit la valeur par défaut de votre compte. Notez que selon la façon dont votre compte/tâche est configuré, la réponse de l'API (comme celle illustrée ci-dessous) inclura widevine:hls ou widevine:dash les sorties, mais pas les deux.

Notez qu'il existe des limites de simultanéité pour PlayReady définies dans votre contrat de service avec Microsoft. Reportez-vous à ce document Microsoft pour plus d'informations sur la gestion de la concurrence. Si la limite de concurrence est dépassée pour une vidéo, elle ne parviendra pas à lire pour les spectateurs au-delà du nombre maximal.

Exemples de création d'emplois en direct

Tous les modes DRM

Voici un exemple de corps de requête pour emballer les sorties avec tous les types de DRM disponibles :

  {
    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 300,
    "ad_insertion": false,
    "static": false,
    "drm":{"modes":["all"]},
    "outputs": [{
        "label": "hls720p",
        "live_stream": true,
        "height": 720,
        "video_bitrate": 2000,

        "live_stream": true,
        "height": 480,

  }

Travail avec WideDevine-HLS

Pour créer des sorties DASH empaquetées par WideDevine (sans obtenir de remplacement de compte pour en faire la valeur par défaut), spécifiez widevine:dash dans le modes comme indiqué dans l'exemple ci-dessous. De même, si votre compte par défaut a été défini pour utiliser Widevine avec DASH, vous pouvez obtenir des sorties WideDevine-HLS pour une tâche particulière en spécifiant widevine:HLS dans le modes.

Exemple de demande

  {
    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 300,
    "ad_insertion": false,
    "static": false,
    "drm":{"modes":["fairplay","widevine:dash"]},
    "outputs": [{
        "label": "hls720p",
        "live_stream": true,
        "height": 720,
        "video_bitrate": 2000,
        "segment_seconds": 6,
        "_keyframe_interval": 60
   },
   {
        "label": "hls480p",
        "live_stream": true,
        "height": 480,
        "video_bitrate": 1000,
        "segment_seconds": 6,
        "_keyframe_interval": 60
   }]
  }
