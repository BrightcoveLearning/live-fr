    Audio multilingue en direct

    Cette rubrique explique comment créer un travail en direct avec plusieurs pistes audio dans différentes langues.

    Exigences

    • Un compte configuré pour l'API Live avec fonction audio multilingue activée (contactez votre responsable de compte pour plus de détails)
    • Une clé API pour l'API Live (fournie lors de la configuration du compte)
    • Encodeur prenant en charge le protocole RTP. La prise en charge audio multilingue est limitée à rtp ou rtp-fec protocole.

    Créez votre travail en direct

    Pour ajouter plusieurs pistes audio à votre travail, une liste de pistes audio doit être fournie dans le corps de la requête lors de la création d'un travail en direct :

    {
  "live_stream": true,
  "region": "us-west-2",
  "protocol": "rtp",
  "cidr_whitelist": [ /* omitted… */ ],
  "outputs": [ /* omitted… */ ],
  "alternate_audio": {
    "tracks": [ { Track Details } ]
  }
}

    Champs pris en charge pour l' Track objet

    Le tableau ci-dessous contient une description complète des champs pris en charge dans l' track objet

    Champs de suivi
    Champ Description
    pid Obligatoire. Le PID à choisir parmi les TS entrants pour la piste spécifiée dans le fichier de flux envoyé à Wowza.
    language Obligatoire. Code de la langue à utiliser. À l'heure actuelle, cela est flexible et peut être dans tous les formats linguistiques exigés/pris en charge par les spécifications, en particulier le RFC5456/BCP47 qui couvre également la norme ISO-639 selon les exigences. https://tools.ietf.org/html/rfc5646

    > LANGUAGE Champ HLS

    > AdaptationSet:lang Champ DASH
    variant Facultatif. Mappe aux valeurs du schéma de rôle DASH : main, alternative, commentaire, supplémentaire, dub. Utilisé dans le rôle DASH urn:mpeg:dash:role:2011. Par défaut, la valeur est main.

    >CARACTÉRISTIQUES HLS voir https://tools.ietf.org/html/rfc8216#section-4.3.4.1
    label Facultatif. Une description textuelle à utiliser pour la piste, préférée pour être utilisée par le joueur dans la mesure du possible. Utilisé dans : > NAME Champ HLS. Par défaut, language avec un ID pour chaque piste, par exemple en-0 s'il n'est pas spécifié.
    streams [] Facultatif. Si la copie n'est pas présente, utilisez uniquement les paramètres audio par défaut.

    HLS > GROUP-ID. (Objet)

    Exemple : { “source”: “720p” } - faire correspondre l'étiquette de la source en sortie.

    Cela imite le streams champ du type de sortie de liste de lecture.
    default Obligatoire sauf lorsque la valeur par défaut Quelle piste audio doit être marquée comme DEFAULT dans le EXT-X-MEDIA ainsi que dans laquelle être muxé dans le flux vidéo *

    S'il n'est pas présent, le premier est par défaut.

    * si plusieurs listes de lecture sont définies avec des valeurs par défaut différentes, la piste par défaut de niveau supérieur est la piste muxée

    Configuration de l'encodeur

    Configurez votre encodeur pour fournir MPEG-TS sur RTP au stream_url à partir de la réponse create. Assurez-vous que les pistes audio fournies dans le corps de la requête sont configurées avec le PID correct dans votre encodeur.

    Test de lecture

    Chargez le playback_url dans Brightcove Player ou Safari pour confirmer la lecture.

    DRM_Lecture

    Actuellement, la DRM n'est prise en charge que par le service Rattrapage (Timeshift). Ajoutez ces champs à votre corps de demande Créer un travail :

    {
  "drm": {
    "modes": [
      "all"
    ]
  },
  "timeshift": {
    "type": "single",
    "startover_duration": 36000,
    "skip_live_output": true
  }
}

    La réponse comprendra des sorties DRM et un timeshift_url à utiliser pour la lecture.
