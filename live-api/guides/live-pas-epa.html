Introduction

Le PAS ou l'EPA peuvent être activés pour une tâche Live afin d'étendre les protections fournies par DRM. PAS fournit un ensemble limité de restrictions : limiter la lecture à un agent utilisateur particulier ou limiter le nombre de modifications d'adresses IP ou d'utilisations totales. Dans le cas Live, il permet également à un client de remplacer le CRT DRM pour cet utilisateur. Ces restrictions sont directement spécifiées dans l'autorisation de lecture JWT.

L'EPA fournit un ensemble beaucoup plus complet de restrictions. Un ensemble de ces restrictions est configuré via l'API Playback Rights Management et est référencé à l'aide d'un droit de lecture id. Cela id peut être associé à la vidéo (à l'aide de l' playback_rights_id attribut illustré ci-dessous) ou inclus dans le JWT. Le deuxième ensemble implique des restrictions au niveau de l'utilisateur telles que les limites de concurrence et le nombre d'appareils. Ceux-ci sont directement spécifiés dans les revendications JWT (qui doivent également inclure un identifiant utilisateur [vuid claim]).

Pour l'un ou l'autre mode, la demande de travail doit inclure un videocloud.video attribut afin qu'une vidéo Video Cloud soit créée et liée à la tâche en direct. L' playback_rights_id attribut peut être mis à jour sur la vidéo Video Cloud pour modifier les restrictions de lecture via l' API CMS.

Vous devrez également créer un JWT avec des revendications appropriées au PAS ou à l'EPA, selon l'utilisation que vous utilisez. Le JWT devra être inclus dans toutes les demandes de serveur de licences. Si vous implémentez votre propre lecteur, vous devrez passer cette valeur dans un BCOV-Auth en-tête ou un paramètre de bcov_auth requête (en plus du jeton DRM).

Créer un emploi Live avec EPA/PAS

Les nouveaux champs suivants sont utilisés pour ajouter de l'EPA et/ou du PAS à une tâche Live :

Nouveaux champs pour l' drm objet :

require_playback_token(booléen)
Active PAS. Lecture Auth JWT sera requise lors de la demande de licence.
check_playback_rights(booléen)
Active l'EPA. Lecture Auth JWT sera requise lors de la demande de licence. Utilise l'identifiant des droits de lecture associé à la vidéo ou à la JWT pour appliquer des restrictions.

Nouveau champ pour l' videocloud.video objet :

Notez que ce champ doit être ajouté à la vidéo à l'aide de l'API CMS après vous créez la tâche en direct, car elle ne peut pas être ajoutée lors de la création de la vidéo.
playback_rights_id(chaîne)
Associe les droits de lecture EPA spécifiés à la vidéo.

Demande d'API CMS

Pour ajouter l'ID des droits de lecture à la vidéo, envoyez une requête de mise à jour vidéo à l'API CMS avec le corps de requête suivant :

{
	"playback_rights_id": "your playback_rights_id"
}
