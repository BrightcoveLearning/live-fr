Gestion des informations d'identification pour l'API Live

Cette rubrique fournit une vue d'ensemble de la gestion des informations d'identification de différents types pour l'API Live.

Introduction

Il existe plusieurs types d'informations d'identification qui peuvent être utilisées conjointement avec vos tâches Live API :

  • Informations d'identification FTP, S3, Video Cloud ou Zencoder dont le système Live a besoin pour pousser des clips vers la destination choisie
  • Informations d'identification RTMP qui empêchent quelqu'un qui obtient l'adresse RTMP de votre vapeur en direct de la lire
  • Informations d'identification OAuth2 ou d'autorisation de base dont le système Live a besoin pour envoyer des notifications à votre gestionnaire de notifications

Les points de terminaison des informations d'identification de l'API Live permettent de créer et de gérer ces informations d'identification, que vous pouvez ensuite référencer par leurs étiquettes lorsque vous créez des travaux de clip en direct ou VOD.

Pour plus de détails sur les opérations de l'API pour les informations d'identification, reportez-vous à la référence de l'API.

Types d'informations d'identification

Le tableau ci-dessous présente les types d'informations d'identification actuellement pris en charge et leurs utilisations.

Types d'informations d'identification
credential_type valeur Description/cas d'utilisation
ftp Informations d'identification pour le site FTP où les clips doivent être envoyés
http Informations d'identification d'autorisation de base pour une adresse http à envoyer des notifications à
oauth Informations d'identification pour l'authentification OAuth2 requises pour l'adresse où les notifications doivent être envoyées
rtmp Ajoutez une authentification au point de terminaison auquel votre encodeur se connecte, afin que toute personne possédant l'URL RTMP ne puisse pas se connecter
s3 Informations d'identification pour un compartiment S3 pour envoyer des clips à
videocloud Informations d'identification OAuth pour votre compte Video Cloud pour envoyer des clips (doivent inclure les autorisations nécessaires pour Dynamic Ingest)
zencoder Authentification pour votre compte Zencoder pour envoyer des clips à

Opérations d'identification

Le tableau ci-dessous présente les points de terminaison pour la gestion des informations d'identification.

Informations d'identification Endpoints
Point de terminaison Méthode Description
/v1/credentials GET Répertorier toutes les informations d'identification du compte
/v1/credentials POST Créer une nouvelle information d'identification pour le compte
/v1/credentials/{credential_id} PUT Mettre à jour les informations d'identification pour le compte
/v1/credentials/{credential_id} DELETE Supprimer une information d'identification pour le compte

Champs d'identification

Le tableau ci-dessous présente les champs des informations d'identification.

Champs d'informations d'identification
Field Type Required Allowed Values Default Description
credential_default_for_type boolean no true | false false Want this credential to be default for the user for this type of credentials?
credential_label string yes     User identifiable name for the credentials
credential_type enum yes ftp | http | oauth | rtmp | s3 | videocloud | zencoder   The type of credential
credential_public string yes     The username, public key, or client id, depending on the credential type
credential_private string yes     The password, private key, or client secret, depending on the credential type
user_id string no   (credentials are available for all account users) user_id (GUID) for which the credentials are being created
