Introduction

Après avoir terminé une tâche, le planificateur enverra une notification au point de terminaison configuré à l'aide de la méthode http PUBLIER avec des détails sur l'action qui a été effectuée et son succès/échec.

Vous pouvez également configurer une pré-notification d'une tâche à venir. Consultez la référence de l'API Live pour plus de détails sur la spécification de l'URL et des paramètres de notification, et notez les différences entre la planification des heures de démarrage/arrêt d'une tâche SEP et pour la planification des clips :

Démarrer / arrêter le planificateur
  • L'URL de notification est définie au niveau supérieur de l'objet du corps de la requête
  • Les notifications sont programmées dans le Activation et désactivation objets (délai en secondes avant l'événement pour envoyer la notification)

Exemple

{
	"job_id": "your Job ID",
	"activation": {
		"time_utc": 1612274100,
		"notification": 600
	},
	"deactivation": {
		"time_utc": 1612281600,
		"notification": 600
	},
	"description": "My Tuesday broadcast",
	"notification_url": "http://example.com/live/jobstartstop/callbacks"
}
Planificateur de clips
  • L'URL de notification est définie au niveau supérieur de l'objet du corps de la requête
  • Les notifications sont programmées dans le agrafe objet (temps en secondes avant que la tâche ne commence à envoyer la notification)

Exemple

{ "job_id": "votre ID de tâche", "description": « Mon clip programmé »,  notification_url » : "http://example.com/live/clip/callbacks", « clip » : { "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879800, "videocloud": { "video": { "name": "My Scheduled Clip" }}}, « time_UTC » : 1612879800,  notification » : 600}}

Organe de notification

Lorsque le serveur envoie une notification à votre notification, la demande inclut un corps JSON. Les champs sont détaillés dans les tableaux ci-dessous.

Champs du corps de la notification
action Nom de l'action. Voir Actions
workflow Mise à jour du flux de travail une fois l'action terminée. La structure dépendra du type de flux de travail.
erreur Si une erreur s'est produite lors de l'exécution de l'action, ce champ contiendra l'erreur. la valeur dépend du type d'action.

Actions

Le tableau suivant détaille les actions par type de workflow.

Actions
Prénom Type de flux de travail Description
notify-activate jobstartstop Attention, votre Job sera bientôt activé.
activate jobstartstop Activation de la tâche terminée. Si "Erreur" est présent, l'activation a échoué et la désactivation sera annulée.
notify-deactivate jobstartstop Attention, votre Job sera bientôt activé.
deactivate jobstartstop Désactivation du travail terminée. Si "Erreur" est présent, la désactivation a échoué.
notify-clip clip Attention, votre clip sera bientôt demandé.
clip clip Demande de clip terminée. Si "Erreur" est présent, la demande a échoué.
