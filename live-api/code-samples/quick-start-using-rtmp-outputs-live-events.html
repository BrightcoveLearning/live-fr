    Brève introduction : Utilisation des sorties RTMP pour les événements en direct

    Ce tutoriel vous guidera à travers la création de sorties RTMP pour mettre en place des événements en direct sur YouTube Live et Facebook Live.

    Présentation

    Ce tutoriel explique au niveau de l'API comment créer un travail de streaming en direct dans Brightcove live , puis créer des sorties RTMP et les rediffuser sur YouTube Live et Facebook Live.

    Remarques :
    • Pour plus d'informations sur l'utilisation des sorties RTMP dans le module Live, voir Utilisation des sorties RTMP en direct pour diffuser sur Facebook et YouTube.
    • Sorties RTMP Est-ce que soutien rtmps urls - vous devez utiliser rtmps si la plateforme sociale le prend en charge.
    • Notez que Facebook a mis fin à la prise en charge de rtmp.streams à partir du 1er mai 2019, pour Facebook, vous DEVEZ utiliser RTMPS.
    • Le seul format actuel pris en charge pour les URL RTMP est rtmps://host:port/application/streamName.

    Ce tutoriel utilisera curl pour faire les requêtes API, mais vous pouvez facilement les faire dans Insomnia , Postman ou d'autres clients REST à la place.

    Créer un travail en direct

    Tout d'abord, nous allons créer un emploi en direct. Vous aurez besoin d'un API-KEY pour l'API Live. Si vous ne le faites pas et que vous êtes intéressé à obtenir l'accès, veuillez contacter votre gestionnaire de compte.

    1. Copiez et collez la curl commande suivante dans un éditeur de texte :

      curl -X POST \
  https://api.bcovlive.io/v1/jobs \
  -H 'Content-Type: application/json' \
  -H 'x-api-key: ' \
  -d '{
    "live_stream": true,
    "region": "",
    "outputs": [{
      "label": "hls360p",
      "live_stream": true,
      "height": 360,
      "video_bitrate": 365,
      "segment_seconds": 6,
      "keyframe_interval": 60
    },
    {
      "label": "hls432p",
      "live_stream": true,
      "height": 432,
      "video_bitrate": 730,
      "segment_seconds": 6,
      "keyframe_interval": 60
    },
    {
      "label": "hls540p",
      "live_stream": true,
      "height": 540,
      "video_bitrate": 2000,
      "segment_seconds": 6,
      "keyframe_interval": 60
    },
    {
      "label": "hls720p3M",
      "live_stream": true,
      "height": 540,
      "video_bitrate": 2000,
      "segment_seconds": 6,
      "keyframe_interval": 60
    },
    {
      "label": "hls720p4.5M",
      "live_stream": true,
      "height": 540,
      "video_bitrate": 4500,
      "segment_seconds": 6,
      "keyframe_interval": 60
    }]
  }'

    2. Remplacer :

      • avec la région disponible la plus proche de votre encodeur, consultez les régions disponibles les plus proches de votre encodeur. Par exemple : us-west-2
      • avec votre clé API live Brightcove. Par exemple : abcdfeg-this-is-a-fake-api-key-FgJajjasd12hJHsZ
    3. Après avoir apporté ces modifications, copiez et collez le code dans Terminal ou toute application de ligne de commande que vous utilisez et exécutez-le.

    4. La réponse devrait être quelque chose comme ceci :

      {
    "id": "3b6871bf2f344acaa6b397d09b476018",
    "outputs": [... removed for simplicity ...],
    "stream_url": "rtmp://ep1-usw2.bcovlive.io:1935/3b6871bf2f344acaa6b397d09b476018",
    "stream_name": "alive",
    "static": false,
    "encryption": {},
    "playback_url": "https://bcovlive-a.akamaihd.net/3b6871bf2f344acaa6b397d09b476018/us-west-2/NA/playlist.m3u8",
    "playback_url_dvr": "https://bcovlive-a.akamaihd.net/3b6871bf2f344acaa6b397d09b476018/us-west-2/NA/playlist_dvr.m3u8"
}

    Ces tâches créeront 5 formats associés basés sur les recommandations Apple.

    Configuration de l'encodeur

    Les étapes indiquées ici supposent un encodeur Elemental Live Box , qui a été utilisé pour tester les étapes de ce tutoriel. Si vous avez un encodeur différent, les paramètres doivent être similaires.

    Les paramètres les plus importants pour ce cas sont :

    • Configurer la source du code temporel comme « horloge système »
    • Réglez « fréquence du code temporel OnFi » sur 1
    • Cochez « insertion du code temporel » dans la vidéo
    • Fortement recommandé : Framerate = suivre la source

    Ajouter le flux en direct à YouTube en direct

    Nous supposons ici que vous avez un compte YouTube activé pour la diffusion en direct.

    1. Cliquez sur Go live, et vous devriez voir :

    2. Cliquez sur « Encoder live streaming » :

    3. Configurez vos métadonnées YouTube en direct et utilisez les données « Encoder configuré » pour connecter le flux en direct de Brightcove à YouTube :

    4. Maintenant que nous avons YouTube en direct configuré, nous allons créer la sortie RTMP pour cela à partir de notre travail Live.

    5. Copiez la curl commande ci-dessous dans un éditeur de texte :

      curl -X POST \
  https://api.bcovlive.io/v1/jobs//rtmpouts \
  -H 'Content-Type: application/json' \
  -H 'x-api-key: ' \
  -d '{
  "url": "rtmp://a.rtmp.youtube.com/live2/"
}'

    6. Remplacer :

      • avec votre identifiant d'emploi en direct Brightcove. Dans notre exemple, cette valeur serait 3b6871bf2f344acaa6b397d09b476018 - votre valeur, à partir de la réponse à votre demande de créer le travail en direct, sera différente.
      • pour votre clé API live Brightcove. Par exemple : abcdfeg-this-is-a-fake-api-key-FgJajjasd12hJHsZ
      • pour le nom de flux /clé que YouTube fournit
    7. Collez le code modifié dans votre application de ligne de commande et exécutez-le.

    8. La réponse devrait être quelque chose comme ceci :

      {
          "connection_info": {
            "host": "a.rtmp.youtube.com",
            "port": 1935,
            "application": "live2",
            "streamName": ""
          },
          "duration_history": [],
          "rtmp_out_id": "27aacdfa5bcf4436ade0519e6a748aba",
          "stream_start": 1542902218238
        }

    9. Pour tester la lecture YouTube, vous pouvez aller sur YouTube ma chaîne:

    Ajouter le flux en direct à Facebook en direct

    Nous supposons ici que vous avez un compte Facebook activé pour la diffusion en direct.

    1. Cliquez sur Live Video

    2. Cliquez sur Connect and Stream Key pour obtenir les données nécessaires pour connecter votre travail en direct Brightcove :

    3. Ensuite, créez une sortie en direct pour le travail en direct Brightcove en utilisant « server url » et « stream key » de Facebook. Copiez la curl commande ci-dessous et collez-la dans un éditeur de texte :

      curl -X POST \
  https://api.bcovlive.io/v1/jobs//rtmpouts \
  -H 'Content-Type: application/json' \
  -H 'x-api-key: ' \
  -d '{
  "url": "rtmp://live-api-s.facebook.com:80/rtmp/"
}'

    4. Remplacer :

      • avec votre identifiant d'emploi en direct Brightcove. Dans notre exemple, cette valeur serait 3b6871bf2f344acaa6b397d09b476018 - votre valeur, à partir de la réponse à votre demande de créer le travail en direct, sera différente.
      • avec votre clé API live Brightcove. Par exemple : abcdfeg-this-is-a-fake-api-key-FgJajjasd12hJHsZ
      • avec votre nom de flux que Facebook vous donne

    5. La réponse devrait être quelque chose comme ceci :

      {
  "connection_info": {
      "host": "live-api-s.facebook.com",
      "port": 80,
      "application": "rtmp",
      "streamName": ""
  },
  "duration_history": [],
  "rtmp_out_id": "59323c3d1e2b44748755853193fb126c",
  "stream_start": 1542904101097
}

    6. Pour tester votre flux Facebook Live, cliquez sur « Go Live » sur votre page Facebook.
