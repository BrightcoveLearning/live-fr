Akamai déprécie l'ingestion RTMP. C'est pourquoi Brightcove cessera de prendre en charge Akamai Live Streaming, tant pour l'entrée RTMP pour l'emballage HLS/DASH que pour les entrées HLS le 1er août 2021. Veuillez contacter l'assistance Brightcove pour en savoir plus sur le basculement et la migration vers Brightcove Live.

Fonctionnalités de Brightcove Live :

  • Brightcove en direct
  • HLS (Clear, DRM et AES-128)
  • DASH (avec PlayReady DRM et Widevine DRM uniquement)
  • Chaînes 24/7
  • Heures de diffusion en direct
  • DRM
  • DVR
La solution de sortie RTMP universelle avec HLS d'Akamai prend uniquement en charge les codecs AAC-LC ou HE-AAC. Le MP3 n'est pas pris en charge dans le cadre d'un flux audio/vidéo ou audio uniquement pour la diffusion HLS.

Cette rubrique vous aidera à diffuser un événement en direct à l'aide d'Akamai Stream Packaging, de l'encodeur Telestream Wirecast, de vidéos de ressources distantes et du Brightcove Player. D'autres encodeurs peuvent également être utilisés. Pour consulter la liste des options de diffusion en direct de Brightcove, lisez la rubrique Diffusion de flux en direct.

Pour diffuser un flux en direct à l'aide d'Akamai Stream Packaging, vous devez :

  • Activer votre compte Video Cloud pour la diffusion en direct
  • Créez une vidéo en direct à l'aide du module Media
  • Configurer Telestream Wirecast pour un événement en direct
  • Générer le code de publication pour un événement en direct

Activation d'une diffusion en direct dans votre compte Video Cloud

Avant toute chose, contactez l'assistance Brightcove et dites-leur que vous souhaitez activer votre compte pour la diffusion en direct. Demandez-leur également d'activer le packaging du flux pour ce compte. Lorsque votre compte est configuré, vous recevrez un e-mail contenant un ensemble d'informations d'identification et d'informations de configuration qui seront nécessaires pour configurer le streaming en direct.

Note : Les informations d'identification d'Akamai HD sont provisionnées géographiquement en fonction de l'adresse IP de votre encodeur. Si vous prévoyez une diffusion depuis un nouvel endroit, indiquez à l'assistance Brightcove l'adresse IP externe interrogée par une commande ping de l'encodeur à ce nouvel endroit. En procédant ainsi, Akamai peut resituer le point d'entrée à l'endroit de plus proche pour réduire la latence et le risque d'interruption en circulant le moins possible sur l'internet public. L'assistance Brightcove doit être avertie bien avant car la modification prend un certain temps à être apportée sur le réseau d'Akamai. Si l'encodeur n'est pas encore à l'emplacement, c'est très bien, tant qu'il existe une adresse IP pingable en externe sur le réseau à utiliser.

Création d'un flux vidéo en direct avec le module Médias

Pour gérer un flux en direct, vous devez créer une vidéo avec des ressources distantes dans votre médiathèque. Les fichiers distants sont des fichiers vidéo que vous conservez localement ou sur un système tiers. Vous envoyez ensuite l’URL du fichier vidéo à Brightcove lors de la création de la vidéo. Les ressources distantes peuvent être créées avec le module Médias ou l'API CMS.

Suivez ces étapes pour créer une ressource vidéo distante à l’aide du module Médias

  1. Ouvrez le module Médias.
  2. Cliquez sur + Ajouter une vidéo à distance dans le panneau de navigation latéral.
  3. Saisissez le nom de la vidéo.
  4. Mettez le type de vidéo sur Live.
  5. Définissez le format sur HLS.
  6. Pour l' URL associée, commencez par la valeur ARL fournie sur votre e-mail du support Brightcove. L'URL ARL sera au format : 
     http://bcliveunivsecure-lh.akamaihd.net/[EVENT_ANGLE_BITRATE]@684348
    Vous devez remplacer les espaces réservés EVENT et ANGLE par des valeurs. BITRATE ne sera pas utilisé. Par exemple, votre URL peut être : 
     http://bcliveunivsecure-lh.akamaihd.net/Live_1@684348

    Insérez un i dans l'URL après la partie akamaihd.net de l'URL :

     http://bcliveunivsecure-lh.akamaihd.net/i/Live_1@684348

    En outre, ajoutez /master.m3u8 à la fin de l'URL. Votre URL de rendu finale doit ressembler à ceci :

    http://bcliveunivsecure-lh.akamaihd.net/i/Live_1@684348/master.m3u8
  7. Collez l'URL formatée dans le champ URL et cliquez sur Ajouter un format associé.
  8. Cliquez sur Save pour enregistrer la vidéo distante.
  9. Vérifiez que la ressource distante s'affiche dans le module Médias. Activez la vidéo distante au besoin.

Une fois que vous avez créé une vidéo distante dans le module Médias, vous pouvez éditer ses métadonnées comme pour toute autre vidéo.

Configuration de Telestream Wirecast pour un événement en direct

Dans cette section, nous configurerons et utiliserons l'encodeur Telestream Wirecast pour prendre en charge la diffusion d'un événement en direct. Telestream Wirecast est une application de bureau qui capture le flux entrant de votre caméra et produit un flux qui peut être diffusé via un CDN. D'autres solutions d'encodage matériel et logiciel existent ; elles peuvent être mieux adaptées à votre stratégie de diffusion de contenus en direct. Vous pourrez télécharger une version d'essai du logiciel Wirecast sur le site internet de Telestream.

Note : Le logiciel Wirecast doit être installé et une caméra doit être connectée à votre ordinateur avant de continuer.

Pour configurer Telestream Wirecast pour un événement en direct, procédez ainsi.

  1. Lancez Wirecast.
  2. Cliquez sur + et sélectionnez Capture vidéo.
  3. Sélectionnez votre appareil photo dans la liste. Cliquez sur Ajouter.
  4. Vérifiez que la source vidéo sélectionnée apparaît dans le volet d'aperçu.
  5. Cliquez sur le bouton fléché () pour que la caméra prenne la photo en direct.
  6. Cliquez sur le bouton de diffusion (). Vous serez invité à entrer les paramètres de sortie.
  7. Renseignez le champ Destination sur RTMP Server. Cliquez sur OK.
  8. Pour l' adresse, utilisez la valeur de point d'entrée principal fournie sur l'e-mail du support Brightcove.
  9. Pour le flux, utilisez la valeur [EVENT_ANGLE_BITRATE] @394443 qui a été utilisée lors de la définition de l'URL de ressource distante. Veillez à utiliser les mêmes valeurs pour EVENT et ANGLE que celles utilisées lors de la création de la ressource distante. Définissez la valeur BITRATE sur le débit défini dans Wirecast.
  10. Cliquez sur Set Credentials...
  11. Saisissez le nom d'utilisateur et le mot de passe fournis dans le courriel de l'assistance Brightcove. Cliquez sur OK.
  12. Cliquez sur OK pour enregistrer les paramètres de sortie.
  13. Cliquez sur le bouton Stream. Le flux en direct doit se lancer.

Génération du code de publication d'un événement en direct

  1. Ouvrez le module Médias.
  2. Sélectionnez la vidéo distante du flux en direct.
  3. Cliquez sur Publier et intégrer.
  4. Sélectionnez un lecteur.
  5. Cliquez sur l'URL du lecteur pour prévisualiser le flux en direct dans un navigateur.

Aide supplémentaire

Si vous n'arrivez pas à diffuser correctement votre vidéo en direct, l'assistance Brightcove est à votre disposition. Vous pouvez créer un cas à cette adresse. Pour vous assurer une réponse rapide, vous trouverez ci-dessous une liste des éléments requis par l'assistance Brightcove pour un traitement optimal.

  • Les symptômes précis du flux. Par exemple, le flux peut ne pas s'afficher du tout, ou il peut s'afficher mais saccader. Il peut aussi se figer lors de la lecture.
  • Est-ce que ce flux fonctionnait correctement auparavant ?
  • L'URL du point d'entrée utilisée par votre encodeur
  • Le nom du flux
  • Le logiciel et le matériel d'encodage utilisé
  • L'URL du lecteur à laquelle vous avez publié l'événement en direct
  • L'ID vidéo de votre ressource distante en direct
  • Les résultats du trace-route de votre encodeur jusqu'à l'hôte du point de publication
