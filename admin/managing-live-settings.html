Gestion des paramètres Live

Dans cette rubrique, vous apprendrez comment gérer les paramètres de Brightcove Live.

Les paramètres Live vous permettent de gérer les ardoises de remplissage et les informations d'identification Live.

Note : Les configurations publicitaires côté serveur utilisées par Brightcove Live sont configurées dans le cadre de l'administration des paramètres publicitaires côté serveur. Pour plus d'informations, reportez-vous à la section Configuration des paramètres publicitaires côté serveur.

Gestion des ardoises de remplissage

Les ardoises de remplissage sont du contenu qui peut être diffusé lorsqu'il y a des lacunes dans un saut de diffusion en direct. Les ardoises peuvent être utilisées pour fournir un message « Be back » ou tout autre contenu que vous souhaitez afficher. Tous les formats d'entrée Zencoder sont pris en charge.

Note : Les ardoises de remplissage peuvent également être gérées à l'aide de l'API Live. Pour plus d'informations, consultez Brightcove Live API : Insertion de publicités côté serveur (SSAI).

Pour ajouter une ardoise de remplissage, procédez comme suit.

  1. Clique le ADMINISTRER> Paramètres en direct lien dans l'en-tête du Studio.
    Note : Vous devez être administrateur de compte pour accéder au menu ADMIN.
  2. Cliquez sur + Ajouter l'ardoise de remplissage.
  3. Saisissez un nom.
  4. Entrez l' URL source de l'ardoise.
  5. Cliquez sur Save. Le processus d'ingestion pour l'ardoise commencera.

Pour supprimer une ardoise de remplissage, cliquez sur le bouton Supprimer () à côté de l'ardoise de remplissage.

Gestion des identifiants

Les informations d'identification sont utilisées à la fois par l'API Live et le module Live. Si vous créez des informations d'identification Live Video Cloud, vous devez d'abord créer des informations d'identification d'authentification API. L'ID client et le secret client créés seront utilisés lors de la création des informations d'identification Live Video Cloud. Lors de la création d'informations d'identification FTP, S3 ou RTMP, une authentification API n'est pas nécessaire.

Si vous créez des informations d'identification S3, FTP ou RTMP, passez à l' étape 5. Pour créer des informations d'identification Live Video Cloud, commencez par créer des informations d'identification API.

  1. Cliquez sur le lien ADMIN dans l'en-tête Studio.
    Note : Vous devez être administrateur de compte pour accéder au menu ADMIN.
  2. Cliquez sur le lien API Authentification. La page Authentification API s'ouvrira et affiche vos enregistrements clients actuels.
  3. Cliquez sur Enregistrer une nouvelle application pour créer une nouvelle information d'identification d'API. Les API suivantes doivent être sélectionnées.
    Note : Pour plus d'informations sur la création d'informations d'identification d'authentification API, consultez Gestion des informations d'identification d'authentification API
  4. Lorsque l'identifiant est créé, enregistrez le identité du client et Secret client valeurs. Ces informations seront nécessaires lors de la création des informations d'identification Live Video Cloud.
  5. Cliquez sur le lien Admin > Live Settings dans l'en-tête Studio.
  6. Cliquez sur + Ajouter des informations d'identification.
  7. Saisissez une étiquette. Il s'agit du nom qui sera utilisé en interne pour faire référence aux informations d'identification.
  8. Sélectionnez un type:
    • FTP - Utilisé pour enregistrer la ressource VOD dans un emplacement FTP (utilisation de l'API)
    • RTMP - Utilisé pour créer des informations d'identification RTMP à utiliser par l'encodeur
    • S3 - Utilisé pour enregistrer la ressource VOD sur un compte Amazon S3 (utilisation de l'API)
    • Video Cloud - Permet d'enregistrer la ressource VOD sur un compte Video Cloud (utilisation de l'API)
  9. Pour la clé publique/nom d'utilisateur, saisissez les éléments suivants selon le type de justificatif :
    • FTP - Entrez le nom d'utilisateur du site FTP
    • RTMP - Entrez le nom d'utilisateur que l'encodeur utilisera pour authentifier
    • S3 - Entrez le nom d'utilisateur du compte S3
    • Video Cloud : entrez l'ID client généré pour les informations d'identification d'authentification de l'API
  10. Pour la clé privée ou le mot de passe, saisissez les éléments suivants selon le type d'informations d'identification :
    • FTP - Saisissez le mot de passe du site FTP
    • RTMP - Entrez le mot de passe que l'encodeur utilisera pour s'authentifier
    • S3 - Saisissez le mot de passe du compte S3
    • Video Cloud : entrez le secret client généré pour les informations d'identification d'authentification API
  11. Cliquez sur Save.

Les informations d'identification apparaîtront dans la liste des informations d'identification. Pour supprimer un identifiant, cliquez sur le bouton Supprimer () à côté de l'identifiant.
