Champs pour les profils d'ingest en direct personnalisés

Cette rubrique fournit une référence pour les champs utilisés dans les profils d'ingest pour Live.

Champs de profil

Propriété Type Description Par défaut
account_id entier Les Nuage vidéo identifiant de compte aucune
name chaîne Le nom du profil doit être unique dans le compte aucun
display_name chaîne Le nom complet du profil devant apparaître dans la page Administrateur Studio - peut être le même que name, mais doit être descriptif pour aider les utilisateurs à comprendre l'utilisation prévue du profil. aucun
description chaîne La description du profil aucun
model_version entier Définissez cette option 1 pour faire apparaître le profil dans le Live Module aucun
renditions gamme Tableau d'objets associés []
packages gamme Tableau de packages DRM - actuellement, le DRM n'est pas pris en charge, il doit donc toujours s'agir d'une baie vide []

Champs de format associé

Les tableaux ci-dessous montrent les champs qui peuvent être utilisés dans le corps de la requête pour mettre à jour les profils d'ingest en direct.

Propriété Type Description Par défaut
audio_bitrate entier débit audio en kbits/s (min : 16, max : 1024) correspondre à la source
audio_codec chaîne Codec audio - aac | mp3 Déterminé par le format ou le profil< - dans la plupart des cas, aac sera le meilleur/td>
decoder_bitrate_cap entier en kbps, le débit maximal alimenté au décodeur via un tampon aucun
decoder_buffer_size entier en kbps, la taille du tampon alimenté au décodeur lors de l'utilisation d'un bitrate_cap aucun
forced_keyframe_rate nombre Force une image-clé toutes les images X, mais autorise toujours des images clés supplémentaires pour la coupe de scène, etc. aucun
format chaîne Format vidéo - ts, mp4, flv aucun
h264_profile chaîne Le H.264 possède trois profils couramment utilisés : Baseline(le plus bas) Main, et High.[1-5] baseline
hls_optimized_ts booléen Segments temporels optimisés pour HLS - s'applique uniquement à la sortie HLS aucun
keyframe_interval entier Nombre maximal d'images entre les images clés 250
keyframe_rate flotte Maximum d'images-clés par seconde keyframe_interval de 250
label chaîne Type d'image pour les rendus d'images - poster ou thumbnail aucun
live_stream booléen si ce sera flux vidéo en direct aucun
live_dvr_sliding_window_duration entier Durée du flux à garder disponible pour la livraison LiveDVR en quelques secondes 300
max_video_bitrate entier en kbits/s, 123 = 123 Kbps - h264 uniquement aucun
max_frame_rate flotte limite le frame_rate plutôt que le définit. Doit être défini sur 30 sauf si votre compte a été activé par le support pour 60 FPS aucun
media_type chaîne video ou audio aucun
video_bitrate entier en kbits/s, 123 = 123 Kbps (min : 64, max : 10000) aucun
video_codec chaîne par exemple h264, ts Déterminé par le format ou le profil
width entier largeur de l'image en pixels aucun
height entier hauteur de l'image en pixels aucun
